## Install

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">
    ```bash
    npm i -g claude-launchpad
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm add -g claude-launchpad
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn global add claude-launchpad
    ```
  </Tab>
</Tabs>

Or run directly without installing:

```bash
npx claude-launchpad
```

## Recommended Flow

### New project

<Steps>
  <Step>
    ### Scaffold
    ```bash
    claude-launchpad init
    ```
    Generates 6 files: CLAUDE.md, TASKS.md, settings.json (with security, hooks, sandbox), .claudeignore, rules, .gitignore. Scores ~93% out of the box.
  </Step>
  <Step>
    ### Diagnose + fix
    ```bash
    claude-launchpad doctor --fix
    ```
    Catches anything init missed and auto-fixes it.
  </Step>
  <Step>
    ### Let Claude perfect it
    ```bash
    claude-launchpad enhance
    ```
    Claude reads your codebase, restructures CLAUDE.md, and fills in architecture, conventions, and guardrails with real content.
  </Step>
  <Step>
    ### Prove it works
    ```bash
    claude-launchpad eval
    ```
    15 scenarios test if Claude actually follows your rules.
  </Step>
</Steps>

### Existing project

<Steps>
  <Step>
    ### See your score
    ```bash
    cd your-project
    claude-launchpad
    ```
    A typical unconfigured project scores ~42%.
  </Step>
  <Step>
    ### Fix the structure
    ```bash
    claude-launchpad doctor --fix
    ```
    Adds security rules, hooks, permissions, sandbox, missing CLAUDE.md sections. Jumps to ~86%.
  </Step>
  <Step>
    ### Let Claude clean it up
    ```bash
    claude-launchpad enhance
    ```
    The fixer appends sections mechanically. Enhance has Claude restructure and rewrite CLAUDE.md into clean, project-specific content.
  </Step>
  <Step>
    ### Prove it works
    ```bash
    claude-launchpad eval
    ```
  </Step>
</Steps>

## Keep your config healthy

`init` runs once. The other three are designed to run again as your project evolves.

| When | Run |
|---|---|
| Added dependencies, changed frameworks, modified hooks | `doctor --fix` |
| Major refactor, new modules, CLAUDE.md feels stale | `enhance` |
| Before a release, after config changes, Claude ignoring rules | `eval` |

<Callout type="info">
  Your codebase changes. Your config should keep up. Run `doctor` periodically. If the score drops, `--fix` and `enhance` bring it back.
</Callout>

## All Commands

| Command | What it does | Runs |
|---|---|---|
| `claude-launchpad init` | Detect stack, generate config, hooks, permissions | Locally |
| `claude-launchpad` | Score your config 0-100, list issues | Locally |
| `claude-launchpad doctor --fix` | Auto-fix issues: hooks, rules, sections, .claudeignore | Locally |
| `claude-launchpad doctor --watch` | Live score that updates when you save config files | Locally |
| `claude-launchpad enhance` | Claude reads your code and completes CLAUDE.md | Via Claude CLI |
| `claude-launchpad eval` | Run Claude against test scenarios, prove config works | Via Claude CLI |

<Callout type="info">
  Commands marked **Locally** run offline with zero API calls. Commands marked **Via Claude CLI** require Claude Code installed and authenticated.
</Callout>
