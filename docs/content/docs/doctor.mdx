The core of the tool. Runs 7 analyzers against your `.claude/` directory and `CLAUDE.md`. An 8th (Memory) is added automatically when agentic-memory is detected.

Low scores signal real problems: conventions ignored, credentials exposed, context lost mid-session, every session starting from scratch. Doctor finds these gaps before they waste your time.

```bash
claude-launchpad doctor
```

## Flags

| Flag | Description |
|---|---|
| `--fix` | Auto-fix: hooks, CLAUDE.md sections, rules, .claudeignore, credentials, sandbox |
| `--fix --dry-run` | Preview what --fix would change without applying |
| `--watch` | Re-runs every second when config files change |
| `--min-score <n>` | Exit code 1 if below threshold (for CI) |
| `--json` | Pure JSON, no banner |
| `-p, --path <dir>` | Run on a different directory |

## Analyzers

| Analyzer | What it catches |
|---|---|
| **Instruction Budget** | Past ~200 lines, Claude stops reliably following rules at the bottom of CLAUDE.md. Doctor warns before you hit that limit. |
| **CLAUDE.md Quality** | Missing sections mean Claude guesses your stack, conventions, and boundaries. Vague instructions ("write good code") waste budget with zero signal. Also checks for a "When Stuck" section so Claude spins up parallel agents after 3 failed attempts instead of retrying the same approach. |
| **Settings** | Missing hooks, deprecated settings, auto-memory conflicts. Without proper settings, Claude runs without guardrails. |
| **Hooks** | Missing SessionStart = Claude forgets where you left off. Missing PostCompact = Claude loses context mid-session. Missing SessionEnd = memories never sync. Missing auto-format = inconsistent code style. |
| **Rules** | Missing BACKLOG.md, dead rule files, stale references. Without structure, deferred features get lost in conversation. |
| **Permissions** | Credential exposure (SSH keys, AWS creds), bypass mode enabled, sandbox disabled. Security gaps that let Claude access what it shouldn't. |
| **MCP Servers** | Invalid transport configs, missing commands/URLs, MCP servers not in `allowedMcpServers` whitelist. `allowedMcpServers` is a security setting that restricts which MCP servers Claude can use -- without it, any server added to `.mcp.json` or settings is auto-trusted. `--fix` populates it from your configured servers. |
| **Memory** (optional) | MCP server config, hooks, autoMemory conflicts, CLAUDE.md guidance, tool permissions. Only runs when agentic-memory is detected. |

## What --fix applies

<Accordions type="multiple">
  <Accordion title="Security">
    - Credential deny rules (~/.ssh/*, ~/.aws/*, ~/.npmrc) so Claude can't read your keys even if it tries to debug
    - Bypass permissions mode disabled so `--dangerously-skip-permissions` can't undo your safety settings
    - Sandbox enabled with failIfUnavailable for subprocess isolation
    - `allowedMcpServers` populated from your defined MCP servers
    - .env added to .claudeignore (defense in depth alongside the hook)
  </Accordion>
  <Accordion title="Hooks">
    - .env file protection hook (blocks reads/writes to .env files, 100% enforced)
    - Auto-format hook (prettier, gofmt, ruff, etc.) so every edit matches your style
    - Force-push protection hook (prevents accidental remote history destruction)
    - PostCompact hook (re-injects TASKS.md after context compression so Claude stays on track mid-session)
    - SessionEnd hook with memory push (memory projects only, auto-syncs memories when session closes)
  </Accordion>
  <Accordion title="CLAUDE.md">
    - Missing sections: Stack, Commands, Architecture, Off-Limits, Session Start, Backlog, When Stuck, Memory & Learnings
    - Attribution migration (includeCoAuthoredBy → attribution object)
  </Accordion>
  <Accordion title="Files">
    - Starter rules file (.claude/rules/conventions.md)
    - BACKLOG.md with priority tier template
    - .claudeignore with language-specific patterns
    - `/lp-enhance` skill at project scope, only if no project/global/legacy install exists
  </Accordion>
</Accordions>

<Callout type="warn">
  Use `--fix --dry-run` to preview changes before applying. This is recommended for projects with existing configurations.
</Callout>

<Callout type="info">
  After `--fix`, use `/lp-enhance` inside Claude Code to restructure your CLAUDE.md. The fixer appends sections mechanically - the skill rewrites them into clean, project-specific content.
</Callout>

## When to re-run

Doctor isn't a one-time setup tool. It's a health check. Re-run it when:

- You add or remove dependencies
- You change frameworks or build tools
- You modify hooks, permissions, or settings manually
- Your score drops after an `enhance` or manual CLAUDE.md edit
- Before a release, to catch config drift

Use `--watch` during active config editing to see your score update in real-time.
