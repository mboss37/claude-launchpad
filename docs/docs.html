Documentation

Complete reference for every command, flag, and use case.

Install

# Install globally
npm i -g claude-launchpad

# Or run directly without installing (downloads temporarily)
npx claude-launchpad

# Or install as a Claude Code plugin
claude plugin install claude-launchpad

init Free

Auto-detect your project and generate Claude Code configuration.

Usage

# Interactive mode
claude-launchpad init

# Accept all defaults (no prompts)
claude-launchpad init --yes

# Set project name explicitly
claude-launchpad init --name "my-project"

Flags

FlagDescription
-n, --nameProject name (auto-detected from directory name)
-y, --yesAccept all defaults without prompting

What it generates

  • CLAUDE.md with detected stack, commands, and essential sections
  • TASKS.md for session continuity
  • .claude/settings.json with auto-format hooks and .env protection
  • .claudeignore with language-specific ignore patterns

When to use

  • Starting a new project with Claude Code
  • Adding Claude Code to an existing project that has no config
  • Regenerating config after major stack changes

doctor Free

Scan your Claude Code configuration and get a score out of 100.

Usage

# Run doctor (also the default command)
claude-launchpad doctor
claude-launchpad

# Auto-fix all deterministic issues
claude-launchpad doctor --fix

# Watch mode — live score on every config change
claude-launchpad doctor --watch

# CI mode — fail if score drops below threshold
claude-launchpad doctor --min-score 80

# JSON output for scripts and CI
claude-launchpad doctor --json

# Combine flags
claude-launchpad doctor --min-score 80 --json

Flags

FlagDescription
--fixAuto-apply deterministic fixes (add hooks, rules, CLAUDE.md sections)
--watchRe-run on every config file change, live score in terminal
--min-score <n>Exit code 1 if score is below threshold (for CI pipelines)
--jsonOutput full results as JSON
-p, --path <path>Project root path (defaults to current directory)

What --fix applies

  • .env file protection hook (blocks reading and writing .env files)
  • Auto-format hook (prettier, gofmt, rustfmt, etc. based on your stack)
  • Force-push protection hook
  • Missing CLAUDE.md sections (Stack, Commands, Architecture, Off-Limits, Session Start)
  • Starter rules file (.claude/rules/conventions.md)

When to use

  • After setting up a new project — check if the config is complete
  • After changing CLAUDE.md or hooks — verify nothing broke
  • In CI — block PRs that degrade config quality
  • With --watch — keep a terminal open while editing configs

enhance Uses Claude

Let Claude read your codebase and complete CLAUDE.md with real content.

Usage

# Launch Claude interactively to enhance CLAUDE.md
claude-launchpad enhance

What it does

Opens an interactive Claude Code session with a pre-loaded prompt. Claude reads your actual codebase and fills in:

  • Architecture — directory structure, data flow, key modules
  • Conventions — naming patterns, imports, state management, API patterns
  • Off-Limits — protected files, anti-patterns, things that should never change
  • Key Decisions — architectural decisions visible in the code

When to use

  • After init generates a skeleton CLAUDE.md with TODOs
  • When doctor reports "Missing Architecture section"
  • When your CLAUDE.md has generic advice instead of project-specific content
  • After major refactors that change the codebase structure

eval Uses Claude

Test if your Claude Code config actually drives correct behavior.

Usage

# Run all 9 built-in scenarios
claude-launchpad eval --suite common

# Single run per scenario (faster, less accurate)
claude-launchpad eval --suite common --runs 1

# 5 runs per scenario (slower, more statistical confidence)
claude-launchpad eval --suite common --runs 5

# Keep sandbox directories for inspection
claude-launchpad eval --suite common --runs 1 --debug

# Custom scenarios from your own directory
claude-launchpad eval --scenarios ./my-scenarios/

# JSON output
claude-launchpad eval --suite common --json

# Longer timeout for complex scenarios
claude-launchpad eval --suite common --timeout 180000

Flags

FlagDescription
-s, --suite <name>Eval suite to run (e.g., "common")
--runs <n>Runs per scenario, median score used (default: 3)
--timeout <ms>Timeout per run in milliseconds (default: 120000)
--scenarios <path>Custom scenarios directory
--debugPreserve sandbox directories for inspection
--jsonJSON output
-p, --path <path>Project root path

Built-in scenarios (9)

ScenarioTests
security/sql-injectionUses parameterized queries, not string concatenation
security/env-protectionCreates .env.example, never writes .env
security/secret-exposureUses environment variables, not hardcoded keys
security/input-validationValidates user input at API boundaries
conventions/error-handlingTry-catch with no empty catch blocks
conventions/immutabilityReturns new objects, never mutates arguments
conventions/file-sizeNo generated file exceeds 800 lines
conventions/no-hardcoded-valuesUses named constants, not magic numbers
workflow/git-conventionsFollows conventional commit format

Cost estimate

Each scenario spawns one Claude session. 9 scenarios x 3 runs = 27 sessions. Estimated cost: $1-3 depending on model. Use --runs 1 for a quick check (~$0.50).

When to use

  • After changing your CLAUDE.md rules — did the change actually help?
  • Comparing two configs — A/B test which one produces better code
  • Before shipping a config to the team — prove it works
  • With --debug — inspect exactly what Claude wrote in the sandbox

Plugin commands

When installed as a Claude Code plugin, use these inside Claude:

/launchpad:init      # same as claude-launchpad init
/launchpad:doctor    # same as claude-launchpad doctor
/launchpad:enhance   # Claude reads your codebase and completes CLAUDE.md
/launchpad:eval      # same as claude-launchpad eval

The plugin also adds a hook that nudges you to re-check your score when you edit config files.
