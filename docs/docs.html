Install

# Install globally
npm i -g claude-launchpad
# Or run directly without installing
npx claude-launchpad

init Free

Auto-detect your project and generate Claude Code config from scratch.

# Interactive mode
claude-launchpad init
# Accept all defaults
claude-launchpad init --yes
# Set project name
claude-launchpad init --name "my-project"
FlagDescription
-n, --nameProject name (auto-detected from directory)
-y, --yesAccept all defaults without prompting

What it generates (6 files)

  • CLAUDE.md — stack, commands, conventions, guardrails, memory management instructions
  • TASKS.md — sprint tracking, session continuity, deferred issues
  • .claude/settings.json — $schema, credential deny rules (~/.ssh, ~/.aws, ~/.npmrc), sandbox enabled, bypass mode disabled, hooks for .env protection + destructive command blocking + auto-format + PostCompact
  • .claude/.gitignore — prevents local settings, plans, memory from being committed
  • .claudeignore — language-specific ignore patterns
  • .claude/rules/conventions.md — language-specific starter rules

doctor Free

Scan your config, score it 0-100, list issues.

# Run doctor (also the default command)
claude-launchpad
# Auto-fix all issues
claude-launchpad doctor --fix
# Live score on config changes
claude-launchpad doctor --watch
# CI: fail if score below threshold
claude-launchpad doctor --min-score 80
# JSON output for scripts
claude-launchpad doctor --json
FlagDescription
--fixAuto-fix: hooks, CLAUDE.md sections, rules, .claudeignore
--watchRe-runs every second when config files change
--min-score <n>Exit code 1 if below threshold (for CI)
--jsonPure JSON, no banner
-p, --path <dir>Run on a different directory

What --fix applies

  • Credential deny rules (~/.ssh/*, ~/.aws/*, ~/.npmrc)
  • Bypass permissions mode disabled
  • Sandbox enabled with failIfUnavailable
  • .env file protection hook (blocks reads and writes)
  • .env added to .claudeignore (defense in depth)
  • Auto-format hook (prettier, gofmt, ruff, etc.)
  • Force-push protection hook
  • PostCompact hook for context re-injection after compaction
  • Missing CLAUDE.md sections (Stack, Commands, Architecture, Off-Limits, Session Start, Memory & Learnings)
  • Starter rules file (.claude/rules/conventions.md)
  • .claudeignore with language-specific patterns

enhance Uses Claude

Claude reads your codebase and completes CLAUDE.md with real content.

claude-launchpad enhance

What it does

  • Opens an interactive Claude session with a pre-loaded task
  • Fills in Architecture, Conventions, Off-Limits, Memory & Learnings, Key Decisions
  • Stays under 120-instruction budget — overflows to .claude/rules/
  • Suggests project-specific hooks (including PostCompact for session continuity) and MCP servers
  • You see Claude working in real-time

eval Uses Claude

Test if Claude actually follows your rules.

# Interactive — pick suite, runs, and model
claude-launchpad eval
# Or pass flags directly
claude-launchpad eval --suite security --runs 1 --model haiku
FlagDescription
--suite <name>security (6), conventions (5), or workflow (4)
--model <model>haiku, sonnet, or opus
--runs <n>Runs per scenario (default 3, median score)
--debugKeep sandbox directories for inspection
--jsonJSON output
--timeout <ms>Timeout per run (default 120000)

Results are saved to .claude/eval/ as structured markdown. Feed them back to Claude to fix failures.

How the sandbox works

Each scenario runs in an isolated temp directory. Your code is never copied — only your Claude Code configuration:

  • .claude/settings.json — your hooks, permissions, and schema
  • .claude/rules/* — all your convention and path-scoped rule files
  • .claudeignore — your ignore patterns
  • A scenario-specific CLAUDE.md with test instructions
  • Seed files from the scenario (e.g. a stub src/api.ts)

Claude runs a task in the sandbox and the results are checked with grep/file assertions. This tests whether your full configuration — rules, hooks, and permissions working together — actually makes Claude behave correctly.
